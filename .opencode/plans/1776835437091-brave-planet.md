# Fix: Register Always Shows Generic Error

## Root Cause

**Primary:** Client-side password validation mismatch between UI and `AuthService`.

- `RegisterScreen` password validator only checks `length >= 8`
- `AuthService.register` calls `InputValidator.validatePassword()` which requires:
  - Minimum 8 characters
  - At least one uppercase letter
  - At least one lowercase letter  
  - At least one digit
- When service validation fails, it throws a plain `Exception('Password harus mengandung huruf besar')`
- `ErrorMapper.mapAuthError` checks `error is ApiError` first → **false** for plain `Exception`
- String pattern checks don't match "huruf besar" → falls to generic fallback: `"Terjadi kesalahan, silakan coba lagi"`

**Secondary:** `ErrorMapper` cannot extract messages from plain `Exception` objects.

## Files to Modify

1. `lib/screens/auth/register_screen.dart` — Update password validator to match `InputValidator` rules so user gets immediate UI feedback before submission.
2. `lib/utils/error_mapper.dart` — Add handling for plain `Exception` messages so service-level validation errors are readable.

## Implementation Steps

### Step 1: Update RegisterScreen Password Validator

In `_buildPasswordField` for the password field, replace the validator:

```dart
validator: (v) {
  if (v == null || v.isEmpty) return 'Password wajib diisi';
  if (v.length < 8) return 'Password minimal 8 karakter';
  if (!RegExp(r'[A-Z]').hasMatch(v)) return 'Password harus mengandung huruf besar';
  if (!RegExp(r'[a-z]').hasMatch(v)) return 'Password harus mengandung huruf kecil';
  if (!RegExp(r'[0-9]').hasMatch(v)) return 'Password harus mengandung angka';
  return null;
}
```

### Step 2: Update ErrorMapper to Handle Plain Exceptions

In `mapAuthError`, after the `error is ApiError` block, add extraction for plain `Exception`:

```dart
// Handle plain Exception messages (e.g., from client-side validation)
if (error is Exception) {
  final match = RegExp(r'^Exception: (.+)$').firstMatch(error.toString());
  if (match != null) {
    return _mapBackendMessage(match.group(1)!.trim());
  }
}
```

Also update `_mapBackendMessage` to handle client validation messages:
- Add patterns for:
  - `'password harus mengandung huruf besar'`
  - `'password harus mengandung huruf kecil'`
  - `'password harus mengandung angka'`
  - `'password wajib diisi'`
